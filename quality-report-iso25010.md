### 1.2 性能効率性 (Performance Efficiency)

#### Hotel Planisphere（静的サイト / GitHub Pages想定）

| ページ / 操作 | P50 | P95 | 閾値 | 判定 |
|--------------|-----|-----|------|------|
| トップページ (index.html) | 12 ms | 2,651 ms | 4,000 ms | ✅ (P95キャッシュ差異あり) |
| プラン一覧 (plans.html + Ajax) | 557 ms | 867 ms | 4,000 ms | ✅ |
| 予約フォーム (reserve.html) | 559 ms | 1,215 ms | 4,000 ms | ✅ |
| 連絡方法切替（UI応答） | - | 37 ms | 500 ms | ✅ |
| 合計金額計算（リアルタイム） | - | 17 ms | 1,000 ms | ✅ |
| 5ページ連続遷移 | - | 3,117 ms | 15,000 ms | ✅ |

> **注記**: トップページのP50=12msはブラウザキャッシュによるもの。初回は約3秒。P95=2651msはキャッシュ有効ケースが混在した結果。

#### Login App（Express.js + PostgreSQL / EC2想定）

| API エンドポイント | P50 | P95 | min | max | 判定 |
|------------------|-----|-----|-----|-----|------|
| POST /api/auth/login | 178 ms | 180 ms | 139 ms | 180 ms | ✅ |
| POST /api/auth/logout | 51 ms | 57 ms | 50 ms | 57 ms | ✅ |
| GET /api/auth/status | 56 ms | 63 ms | 53 ms | 63 ms | ✅ |
| POST /api/auth/change-password | 52 ms | 54 ms | 48 ms | 54 ms | ✅ |

> **想定負荷根拠**: じゃらん類似サービス月間1,368万UU → 時間当たり約19,000UU → 推定同時接続200〜500。ピーク時は3〜5倍（1,000〜2,500同時接続）。Login APIはbcrypt処理込みで178ms以下は十分実用的。
