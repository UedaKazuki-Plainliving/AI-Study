# HAYST法 テスト設計書 — 社内Webシステム ログイン・ユーザー管理機能

**手法**: HAYST法（Hierarchical Aspects and Story-based Testing / 秋山浩一氏提唱）  
**特徴**: テスト対象を「因子（パラメータ）」と「水準（値のバリエーション）」に分解し、組み合わせ表（全組み合わせ or ペアワイズ）でテストを体系的に生成する。ストーリーテストで疎な利用法もカバーする。

---

## 1. ログイン機能 因子・水準表

### 因子定義

| 因子ID | 因子名 | 説明 |
|--------|--------|------|
| F1 | ユーザーID入力値 | フォームに入力するID |
| F2 | パスワード入力値 | フォームに入力するPW |
| F3 | ユーザー登録状態 | DB上のユーザーの状態 |
| F4 | パスワード一致状態 | 入力PWとDB上のハッシュの一致 |
| F5 | ログイン試行回数 | 過去の失敗累積回数 |
| F6 | パスワード期限状態 | password_changed_atの状態 |

### 水準定義

| 因子 | 水準1 | 水準2 | 水準3 | 水準4 |
|------|-------|-------|-------|-------|
| F1: ユーザーID入力値 | 空 | 半角英数字（有効） | 記号含む（無効） | 21文字超（無効） |
| F2: パスワード入力値 | 空 | 8文字以上（有効） | 7文字以下（無効） | — |
| F3: ユーザー登録状態 | 未登録 | 有効 | ロック中 | 無効 |
| F4: パスワード一致状態 | 一致 | 不一致 | — | — |
| F5: ログイン試行回数 | 0回 | 1〜4回 | 5回（ロック境界） | — |
| F6: パスワード期限状態 | 有効（<90日） | 期限切れ（≥90日） | — | — |

### 禁則条件（実際には起こり得ない組み合わせ）

| 禁則ID | 組み合わせ | 理由 |
|--------|-----------|------|
| 禁-1 | F1=空 × F3=有効 | 空IDはバリデーションで弾かれサーバーに届かない |
| 禁-2 | F1=記号含む × F3=有効 | 不正IDはサーバーバリデーションで弾かれる |
| 禁-3 | F3=未登録 × F4=一致 | 未登録ユーザーのPWは存在しない |
| 禁-4 | F3=ロック中 × F5=0回 | ロック状態はF5≥5が前提 |
| 禁-5 | F2=空 × F4=一致 | 空PWは一致しない |

## 5. ストーリーテスト（因子・水準でカバーできないシナリオ）

HAYST法では因子・水準の組み合わせに加えて、**ストーリー**（実際の利用場面）もテストケースとして追加する。

### ストーリー1: 年度末の一斉アカウント処理

```
背景: 3月末に退職・異動が集中する
1. 太郎が管理画面を開く
2. 退職者10名を順に無効化する
3. 異動者5名のパスワードをリセットする（PW変更要求）
4. 新入社員15名のアカウントを作成する
5. 各操作後に一覧が正しく更新されていることを確認する
期待: 全操作が正常に完了し、一覧に反映される
```

### ストーリー2: 連続ロック→解除→即ログイン

```
1. 花子が5回連続でPWを誤入力 → アカウントロック
2. 花子からIT部門へ連絡
3. 太郎が管理画面でロック解除
4. 花子が正しいPWで即座にログイン成功
5. 失敗カウントがリセットされていることを確認
期待: ロック解除後すぐにログイン可能
```

### ストーリー3: パスワード変更後の連続利用

```
1. 花子のPWが期限切れ（90日経過）
2. ログイン → PW変更画面遷移
3. 新PW "NewSpring2024!" を設定
4. 2秒後にログイン画面へ遷移
5. 新PWでログイン成功
6. 翌日も新PWでログイン成功（セッション新規）
期待: 変更後のPWで継続ログイン可能
```
