# ゆもつよメソッド テスト設計書 — 社内Webシステム ログイン・ユーザー管理機能

**手法**: ゆもつよメソッド（湯本剛氏提唱）  
**特徴**: ペルソナのユーザーストーリーから観点を導出し、「疎な機能利用」を含む目的機能の広がりを考慮しながら、リスクベースでテストを優先付けする。

## 1. ペルソナ定義

### ペルソナA: 田中 花子（一般社員・30代女性）

| 項目 | 内容 |
|------|------|
| 役職 | 営業部 一般社員 |
| ITスキル | 基本的なPC操作は問題なし。システム障害時はIT部門に電話する |
| 利用頻度 | 毎日出社時に1回ログイン。週3回以上システムを利用 |
| 課題・懸念 | パスワードを忘れがち。長期休暇後にロックされてIT部門に電話することが多い |
| ゴール | ストレスなくログインして業務に集中したい |

### ペルソナB: 山田 太郎（IT部門管理者・40代男性）

| 項目 | 内容 |
|------|------|
| 役職 | IT部門 システム管理者 |
| ITスキル | 高い。DB操作・サーバー管理の経験あり |
| 利用頻度 | 月3〜5回。主に人事異動・インシデント対応時 |
| 課題・懸念 | 退職者アカウントの削除漏れ・ロック解除の誤操作 |
| ゴール | 素早く・正確にアカウント状態を管理したい |

## 6. 目的機能の考察（疎な利用法）

ユーザーが「意図せず」組み合わせる機能や、想定外の利用パターンを考慮する。

| シナリオID | 考察シナリオ | ユーザーの行動 | テスト観点 |
|-----------|------------|-------------|---------|
| PF-01 | 長期休暇後の初ログイン | 花子が3ヶ月ぶりにログイン → PW期限切れ + 変更画面 → 新PW設定 → 再ログイン | 期限切れ遷移〜変更〜再認証の連続フロー |
| PF-02 | ロック後のIT部門対応 | 花子が5回失敗 → IT部門（太郎）が管理画面でロック解除 → 花子が再ログイン | ロック→管理解除→再認証の一連フロー |
| PF-03 | 退職者処理 | 太郎が退職者を無効化 → 退職者が無効化後にログイン試行 | 無効化後のログイン拒否確認 |
| PF-04 | 新入社員のファーストログイン | 太郎がアカウント作成時にPW変更要求セット → 初ログインで変更画面遷移 | 管理者設定〜初回ログインフロー |
| PF-05 | ブラウザバック攻撃 | ログアウト後にブラウザの「戻る」ボタンで前ページへ移動 | セッション破棄の確認 |
| PF-06 | 複数タブ操作 | タブAでログイン中、タブBでログアウト → タブAで操作継続 | セッション無効後のAPI応答 |

## 9. テストケース詳細（優先度上位）

### TC-Y50: 長期休暇後の全フロー

**目的**: 期限切れ検知〜PW変更〜再ログインの一連フローが正常に機能することを確認  
**ペルソナ**: 田中 花子（長期休暇明け）  
**前提条件**: ユーザー `hanatanaka` が存在し `forcePasswordChange: true` が設定されている

| ステップ | 操作 | 期待結果 |
|---------|------|---------|
| 1 | ログイン画面を開く | ログイン画面が表示される |
| 2 | hanatanaka / TestP@ss1 を入力してログイン | PW変更画面に遷移する |
| 3 | `.pw-expired-notice` の確認 | 「パスワードの有効期限が切れています」的なメッセージが表示される |
| 4 | 新PW `NewFlower2024!` / 確認PW `NewFlower2024!` を入力して変更 | 「パスワードを変更しました」が表示される |
| 5 | 2秒待機 | ログイン画面へ遷移する |
| 6 | hanatanaka / NewFlower2024! でログイン | ホーム画面に遷移する |

### TC-Y52: 退職者無効化後のログイン拒否

**目的**: 無効化されたアカウントがログインできないことを確認  
**ペルソナ**: 太郎（退職者処理）、退職した社員（不正アクセス試行者）  

| ステップ | 操作 | 期待結果 |
|---------|------|---------|
| 1 | 管理画面で退職者 `retired001` を「無効化」する | badge-inactive 表示 |
| 2 | ログイン画面で `retired001` / 正しいPW でログイン試行 | 認証エラーが表示される（ログイン拒否） |
| 3 | API: POST /api/auth/login (retired001) | 401 AUTH_FAILED |
