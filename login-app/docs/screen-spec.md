## SCR-001 ログイン画面

### 画面概要

ユーザーIDとパスワードを入力し、認証を行う。

### レイアウト

```
┌────────────────────────────────┐
│   PLAIN LIVING Co., Ltd.       │
│      社内Webシステム            │
├────────────────────────────────┤
│  [認証エラーメッセージ領域]      │
│                                │
│  ユーザーID ＊                  │
│  [入力欄                    ]  │
│  ※半角英数字　1〜20文字         │
│  [フィールドエラーメッセージ]    │
│                                │
│  パスワード ＊                  │
│  [入力欄                    ]  │
│  ※半角英数記号　8〜32文字       │
│  [フィールドエラーメッセージ]    │
│                                │
│  [    ログイン    ]             │
└────────────────────────────────┘
┌────────────────────────────────┐
│ 🧪 テスト制御パネル（折りたたみ）│
└────────────────────────────────┘
```

### 入力項目

#### ユーザーID

| 項目 | 内容 |
|------|------|
| 入力タイプ | text |
| 最大文字数 | 20文字（maxlength属性） |
| プレースホルダー | 「例: user001」 |
| autocomplete | username |

**バリデーション**

| 順序 | 条件 | エラーメッセージ |
|------|------|----------------|
| 1 | 未入力 | ユーザーIDを入力してください |
| 2 | 半角英数字以外を含む | ユーザーIDは半角英数字で入力してください |
| 3 | 1〜20文字の範囲外 | ユーザーIDは1〜20文字で入力してください |

- バリデーション実施タイミング：フォーカスアウト時 / ログインボタン押下時
- 正規表現: `/^[a-zA-Z0-9]+$/`

#### パスワード

| 項目 | 内容 |
|------|------|
| 入力タイプ | password（マスク表示） |
| 最大文字数 | 32文字（maxlength属性） |
| autocomplete | current-password |

**バリデーション**

| 順序 | 条件 | エラーメッセージ |
|------|------|----------------|
| 1 | 未入力 | パスワードを入力してください |
| 2 | 半角英数記号以外を含む | パスワードは半角英数記号で入力してください |
| 3 | 8〜32文字の範囲外 | パスワードは8〜32文字で入力してください |

- バリデーション実施タイミング：フォーカスアウト時 / ログインボタン押下時
- 使用可能記号: `!"#$%&'()*+,-./:;<=>?@[\]^_{|}~` およびバッククォート

### ボタン

#### ログインボタン

| 項目 | 内容 |
|------|------|
| ラベル | ログイン |
| 動作 | クリック時・Enterキー押下時に認証処理を実行 |

### 認証処理フロー

```
ログインボタン押下
      ↓
フィールドバリデーション（フロントエンド）
      ↓ エラーあり → フィールドエラー表示、処理停止
      ↓ エラーなし
POST /api/auth/login（サーバー側処理）
  → アカウントロックチェック
       ロック中 → 認証エラーメッセージ表示（ロック・残り時間）
  → ID・パスワード照合
       不一致 → 失敗回数インクリメント
                5回未満 → 認証エラーメッセージ表示（残り回数付き）
                5回以上 → ロック設定 → 認証エラーメッセージ表示（ロック）
       一致   → パスワード期限チェック
                期限切れ  → SCR-003（パスワード変更画面）へ遷移
                期限内    → SCR-002（ホーム画面）へ遷移
```

### エラーメッセージ表示

#### フィールドエラー

- 表示位置: 各入力欄の直下
- 表示色: 赤（#e53e3e）
- 入力欄の枠色: 赤に変更・背景色 #fff5f5

#### 認証エラー（共通エリア）

- 表示位置: 入力欄の上部
- 認証失敗時の背景色: 薄赤（#fff5f5）・枠 #feb2b2
- ロック時の背景色: 薄黄（#fffbeb）・枠 #f6ad55

| ケース | メッセージ |
|--------|-----------|
| 認証失敗（残り4回） | ユーザーIDまたはパスワードが正しくありません。（あと4回失敗するとロックされます） |
| 認証失敗（残り1回） | ユーザーIDまたはパスワードが正しくありません。（あと1回失敗するとロックされます） |
| ロック直後 | ログインに連続5回失敗したため、アカウントをロックしました。30分後に自動解除されます。 |
| ロック中（再アクセス） | アカウントがロックされています。約28分30秒後に自動解除されます。 |

- ユーザーが入力を開始すると認証エラーメッセージは非表示になる

### テスト制御パネル

> **【API版では廃止】**
> テスト制御パネルは localStorage を操作するモック版専用機能のため、API版（PostgreSQL連携）では実装しない。
> テスト用の状態操作（ロック設定・パスワード期限切れ）は管理画面（admin.html）から行うこと。

## SCR-003 パスワード変更画面

### 画面概要

パスワードの有効期限が切れた場合に強制表示される変更画面。

### レイアウト

```
┌────────────────────────────────┐
│  パスワードの変更               │
│  ⚠️ パスワードの有効期限（90日）│
│     が切れています。新しいパス  │
│     ワードを設定してください。  │
├────────────────────────────────┤
│  [成功メッセージ（非表示→表示）] │
│                                │
│  新しいパスワード ＊            │
│  [入力欄                    ]  │
│  ※半角英数記号　8〜32文字       │
│  [フィールドエラーメッセージ]    │
│                                │
│  新しいパスワード（確認） ＊    │
│  [入力欄                    ]  │
│  [フィールドエラーメッセージ]    │
│                                │
│  [パスワードを変更する]         │
└────────────────────────────────┘
```

### 入力項目

#### 新しいパスワード

| バリデーション | エラーメッセージ |
|--------------|----------------|
| 未入力 | パスワードを入力してください |
| 半角英数記号以外 | パスワードは半角英数記号で入力してください |
| 8〜32文字範囲外 | パスワードは8〜32文字で入力してください |

#### 新しいパスワード（確認）

| バリデーション | エラーメッセージ |
|--------------|----------------|
| 未入力 | 確認用パスワードを入力してください |
| 新しいパスワードと不一致 | パスワードが一致しません |

### パスワード変更処理フロー

```
「パスワードを変更する」押下
      ↓
バリデーション（新PW + 確認PW）
      ↓ エラーあり → フィールドエラー表示、処理停止
      ↓ エラーなし
POST /api/auth/change-password（サーバー側でDBのパスワードハッシュ・更新日時を書き換え）
      ↓
成功メッセージ表示
ボタン無効化
2秒後 → SCR-001（ログイン画面）へ遷移
```

### メッセージ

| 種別 | メッセージ |
|------|-----------|
| 成功 | ✅ パスワードを変更しました。ログイン画面に戻ります。 |

## 共通仕様

### バリデーション実施タイミング

| タイミング | 動作 |
|-----------|------|
| フォーカスアウト（blur） | 該当フィールドのみバリデーション実行 |
| ボタン押下 | 全フィールドのバリデーションを実行 |
| 入力中（input） | 認証エラーメッセージのみ非表示（フィールドエラーは残す） |

### 状態管理

失敗回数・ロック状態・パスワード期限はすべてサーバー側 DB（PostgreSQL の `users` テーブル）で管理する。
ブラウザの localStorage は使用しない。

### テスト用認証情報

| 項目 | 値 |
|------|-----|
| ユーザーID | user001 |
| パスワード | P@ssword |
